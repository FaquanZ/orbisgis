Services in OrbisGIS

This is the description of the available services in OrbisGIS. An OrbisGIS service is just a java instance that is accessible by a name and that implements a specified interface. From a practical point of view, services will be the entry point to access the different functionalities in OrbisGIS

Service nameService interfaceDescription
org.orbisgis.BackgroundManagerorg.orbisgis.pluginManager.background.BackgroundManagerExecute tasks in background processes, showing progress bars. Gives access to the job queue
org.orbisgis.DataManagerorg.orbisgis.DataManagerAccess to the sources, to its properties (indexes, etc.) and its contents, either raster or vectorial
org.orbisgis.EditorManagerorg.orbisgis.views.editor.EditorManagerGets access to the active editor and its document
org.orbisgis.ErrorManagerorg.orbisgis.errorManager.ErrorManagerNotification of errors to the system
org.orbisgis.ExtendedWorkspaceorg.orbisgis.ExtendedWorkspaceGives access to directories inside the workspace. You can use the temporal folder in the workspace through this service. It lets the access to the results folder
org.orbisgis.MapContextManagerorg.orbisgis.editors.map.MapContextManagerGives access to the current MapContext
org.orbisgis.PluginManagerorg.orbisgis.pluginManager.PluginManagerGives access to the plug-in system manager. It can stop the application, access the log files, etc.
org.orbisgis.UIManagerorg.orbisgis.windows.mainFrame.UIManagerGets access to the main frame
org.orbisgis.ViewManagerorg.orbisgis.view.ViewManagerOpen, close and gets access to the views
org.orbisgis.Workspaceorg.orbisgis.pluginManager.workspace.WorkspaceChange workspace, save files in the workspace, etc.
